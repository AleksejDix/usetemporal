### VueTemporal Interface Definition

**Create src/types.ts:**
```typescript
import type { Ref, ComputedRef } from "@vue/reactivity";
import type { Adapter, Period } from "@allystudio/usetemporal";

/**
 * Vue-specific Temporal interface with reactive state
 * This replaces the old Temporal interface which was removed from core
 */
export interface VueTemporal {
  adapter: Adapter;
  weekStartsOn: number;
  browsing: Ref<Period>;  // Reactive browsed period
  now: Ref<Period> | ComputedRef<Period>;  // Reactive current time
}

/**
 * Options for creating a Vue temporal instance
 */
export interface UseTemporalOptions {
  date: Date | Ref<Date>;
  now?: Date | Ref<Date>;
  adapter: Adapter;
  weekStartsOn?: number;
}

/**
 * Builder interface - convenience methods that wrap pure operations
 * Extends VueTemporal to include reactive state AND convenience methods
 */
export interface TemporalBuilder extends VueTemporal {
  period(date: Date, unit: Unit): Period;
  period(options: { start: Date; end: Date }): Period;
  divide(period: Period, unit: AdapterUnit): Period[];
  merge(periods: Period[], targetUnit?: AdapterUnit): Period | null;
  next(period: Period, count?: number): Period;
  previous(period: Period, count?: number): Period;
  go(period: Period, count: number): Period;
  split(period: Period, date: Date): [Period, Period];
  contains(period: Period, dateOrPeriod: Date | Period): boolean;
  isSame(period1: Period, period2: Period, unit: AdapterUnit | 'custom'): boolean;
}

// Re-export core types for convenience
export type { Period, Unit, AdapterUnit, Adapter } from "@allystudio/usetemporal";
```

### README.md Example Content

```markdown
# @allystudio/usetemporal-vue

Vue.js integration for useTemporal with reactive composables.

## Installation

\`\`\`bash
npm install @allystudio/usetemporal-vue @allystudio/usetemporal
\`\`\`

## Basic Usage

\`\`\`typescript
import { ref } from 'vue'
import { useTemporal, usePeriod } from '@allystudio/usetemporal-vue'
import { createNativeAdapter } from '@allystudio/usetemporal/native'

// Create temporal instance
const temporal = useTemporal({
  adapter: createNativeAdapter(),
  date: ref(new Date())
})

// Create reactive period
const month = usePeriod(temporal, 'month')

// Use in template
console.log(month.value.start) // Reactive!
\`\`\`

## Reactive Adapter Settings

\`\`\`typescript
import { ref, computed } from 'vue'
import { useTemporal } from '@allystudio/usetemporal-vue'
import { createNativeAdapter } from '@allystudio/usetemporal/native'

// Reactive week start setting
const weekStartsOn = ref(1) // Monday

const temporal = useTemporal({
  adapter: computed(() =>
    createNativeAdapter({ weekStartsOn: weekStartsOn.value })
  ),
  date: ref(new Date())
})

// Change setting - everything recalculates!
weekStartsOn.value = 0 // Sunday
\`\`\`

## API

### useTemporal(options)

Creates a reactive temporal instance.

**Options:**
- `adapter: Adapter` - Date adapter (required)
- `date: Date | Ref<Date>` - Initial date
- `now?: Date | Ref<Date>` - Current time reference
- `weekStartsOn?: number` - Week start day (0-6)

**Returns:** `TemporalBuilder` with reactive `browsing` and `now` refs

### usePeriod(temporal, unit)

Creates a reactive period that updates with temporal.browsing.

**Parameters:**
- `temporal: VueTemporal` - Temporal instance
- `unit: Unit | Ref<Unit>` - Period unit type

**Returns:** `ComputedRef<Period>`

## Migration from v2.0.0-alpha.1

\`\`\`typescript
// Before (alpha.1)
import { createTemporal, usePeriod } from '@allystudio/usetemporal'

import { useTemporal, usePeriod } from '@allystudio/usetemporal-vue'
import { createNativeAdapter } from '@allystudio/usetemporal/native'

// Function renamed: createTemporal → useTemporal
const temporal = useTemporal({
  adapter: createNativeAdapter(),
  date: new Date()
})
\`\`\`
```

### Dependencies Explained

**Runtime Dependencies:**
- `@vue/reactivity: ^3.5.18` - Vue's reactivity system (Ref, computed, effect)
- `@allystudio/usetemporal: workspace:*` - Core operations and types

**Why NOT depend on full Vue?**
- Vue package is large (~500KB)
- @vue/reactivity is small (~50KB) and sufficient for our needs
- Allows usage outside Vue SFC context (Node.js, workers, etc.)
- Peer dependency on vue ^3.0.0 ensures compatibility

### Important Implementation Notes

1. **DO NOT delete .removed files from core** - They're preserved for reference and potential future use

2. **Rename createTemporal to useTemporal** - Better reflects Vue composable naming convention

3. **weekStartsOn is an adapter setting** - Not a reactive property of VueTemporal. To change it, create a new adapter:
   ```typescript
   const weekStartsOn = ref(1)
   const temporal = useTemporal({
     adapter: computed(() => createNativeAdapter({ weekStartsOn: weekStartsOn.value })),
     date: new Date()
   })
   ```

4. **Builder pattern is included** - Provides convenience methods but tree-shakable

5. **All exports are tree-shakable** - Unused re-exported operations add 0KB to bundle

6. **Tests must use TZ=UTC** - Critical for date operation consistency
